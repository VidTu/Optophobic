# Optophobic

Скройте "Обновления света отключены" из Sodium Extra.

- Fabric Loader
- Minecraft 1.21.11
- [Sodium](https://modrinth.com/mod/sodium)
- [Sodium Extra](https://modrinth.com/mod/sodium-extra)

## О проекте

Когда вы отключаете "Обновления света" в моде Sodium Extra, он
добавляет надпись "Обновления света отключены" без способа её скрыть.
Этот мод скрывает ту надпись.

## ЧаВО (FAQ)

### Для игроков

**В**: Мне нужна помощь, у меня есть вопросы, я хочу связаться с разработчиком.  
**А**: Можете зайти на [сервер Discord](https://discord.gg/Q6saSVSuYQ).
(главный разработчик говорит по-русски)

**В**: Где я могу скачать этот мод?  
**А**: На [GitHub Releases](https://github.com/VidTu/Optophobic/releases).
Нестабильные версии можно скачать на
[GitHub Actions](https://github.com/VidTu/Optophobic/actions).
Для них потребуется аккаунт GitHub.

**В**: Какие загрузики модов поддерживаются?  
**А**: Поддерживается Fabric. Quilt тоже должен работать.

**В**: Какие версии Minecraft поддерживаются?  
**А**: Minecraft 1.21.11.

**В**: Планируется ли поддержка NeoForge/Forge или других версий?  
**А**: По мере выхода новых версий, я планирую добавлять поддержку для них для
Fabric. Поддержка NeoForge/Forge и старых версий не планируется.

**В**: Нужно ли мне ставить Fabric API или Quilt Standard Libraries?  
**А**: Нет.

**В**: Этот мод нужно ставить на клиент или на сервер?  
**А**: Только на клиент. Sodium и Sodium Extra тоже нужно
ставить только на клиент. Серверной версии не существует.

**В**: Я нашёл баг.  
**А**: Отправляйте все баги [сюда](https://github.com/VidTu/Optophobic/issues)
на английском языке (пожалуйста). Если вы не уверены, баг это или нет, вы можете
зайти в [Discord](https://discord.gg/Q6saSVSuYQ). На уязвимости в моде можно
пожаловаться [сюда](https://github.com/VidTu/Optophobic/security).

**В**: Можно я закину это в свою сборку?  
**А**: Конечно. За упоминание (например, ссылкой на GitHub-страницу мода)
будем премного благодарны, но это необязательно. Монетизация и
распространение модпака разрешены на условиях
[Apache 2.0 License](https://github.com/VidTu/HCsCR/blob/main/LICENSE).

**В**: Почему этот мод не на Modrinth или CurseForge?  
**А**: Я думаю, что это временный "костыль" и разработчик Sodium Extra
должен добавить такую функциональность в его мод через время.
Буду же я не прав - залью на обе платформы.

**В**: Зачем?  
**А**: Мне не нравится, что вообще нельзя скрыть "Обновления света отключены".
Я [попросил](https://github.com/FlashyReese/sodium-extra/issues/507)
разработчика добавить возможость делать это, теперь жду ответа.

**В**: Этот мод повлияет на производительность? Насколько сильно он лагает?  
**А**: Не лагает. Это мод на один миксин, который отключает надпись.

### Для разработчиков

**В**: У этого мода открытый исходный код?  
**А**: [Да.](https://github.com/VidTu/Optophobic) (Доступный под лицензией
[MIT License](https://github.com/VidTu/Optophobic/blob/main/LICENSE))

**В**: Чё так много ненужного базара в этом README?  
**А**: ~~Я заплатил за всю LLM, я буду использовать всю LLM.~~
Писать README (даже без AI, его я использую только для грамматики)
проще, чем писать нормальный код.

**В**: Используется ли AI/LLM/Code Generation/Copilot/etc.?  
**А**: Кроме вышеупомянутых инструментов на проверку грамматики, нет. Я
использую самые лаганные инспекции IntelliJ IDEA, основанные на самых
забагованных алгоритмах. Не могу сказать это за контрибьюторов, но большинство
кода пишет автор проекта. А ещё, мне без разницы на использование AI.

**В**: У Optophobic есть публичное API?  
**А**: Неа. Никаких публичных API в моде нет.
По этой причине, все классы/пакеты дополнительно помечены
[@ApiStatus.Internal](https://javadoc.io/static/org.jetbrains/annotations/26.0.2/org/jetbrains/annotations/ApiStatus.Internal.html).

**В**: А я могу забить и всё равно использовать этот мод при компиляции?
Что насчёт системы версий [SemVer](https://semver.org/lang/ru/) в этом моде?  
**А**: Можете, на свой риск. Система версий SemVer используется для простоты,
но она используется не по задумке. У мода нет публичного API, и поэтому
несовместимые изменения исходников/бинарников могут и будут происходить
даже между минорными и патч версиями.

## Разработка

### Сборка (Компиляция)

Для сборки мода из исходного кода:

1. Требуется 1 ГБ свободной RAM, 1 ГБ свободного места
   на диске и активный доступ к интернету.
2. Установите Java 25 (для Gradle; для компиляции так же будет нужна
   8, скачайте любую, другая будет автоматически скачана через
   Java toolchains) и закиньте её в `PATH` и (или) `JAVA_HOME`.
3. Выполните в терминале/PowerShell команду `./gradlew assemble`.
4. Заберите JAR-ник из папки `./build/libs/`.

### Разработка/Отладка/Дебаггинг

Тут буквально нечего запускать, потому что тут нет какого-либо плагина Minecraft
для Gradle. Все компилируется и тестируется в стороннем лаунчере. Да, я знаю,
что это шляпа. Это мод для нескольких версий, нескольких загрузчиков и это всё
в одном JAR-файле. Нет другого разумного способа это нормально реализовать.
Этот способ тоже не очень то и нормальный. Удачи!

Рекомендуемая IDE для разработки - IntelliJ IDEA (Community или Ultimate)
с плагином Minecraft Development. тем не мене, это не строгое требование.
Любая IDE/редактор будет работать нормально.
